# Security Policy

## Supported Versions

This repository contains code from events, hackathons, and school projects. These projects are generally provided "as-is" for archival and portfolio purposes. Active maintenance is limited to the most recent versions of these projects.

| Version | Supported          |
| ------- | ------------------ |
| Latest Main/Master | :white_check_mark: |
| Archived / School Projects | :x:                |
| Event / Hackathon Tags | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability—such as accidentally exposed credentials, API keys, or sensitive data within this archive—please report it responsibly.

### How to report
1. **Do not open a public issue.** Please report any findings privately to avoid public exploitation.
2. Send an email to **aditkpython@gmail.com** with the subject line: `Security Vulnerability - [Directory Name]`.
3. Please include a brief description of the issue and the specific location (file and line number) where it was found.

### What to expect
* **Acknowledgment:** As an individual developer, I will attempt to respond to your report within **1 day**.
* **Remediation:** If the vulnerability involves sensitive data or secrets, I will prioritize scrubbing the Git history or rotating credentials immediately.
* **Disclosure:** Since these are personal projects, no formal CVEs will be issued. However, I am happy to credit you for your discovery in the remediation commit.
